## 1. Безопасность финтеха

### Что делают лидеры рынка

Финтех-приложения мирового уровня (Revolut, Monarch Money, Copilot Money) внедряют многоуровневую систему безопасности:

- **Шифрование данных**: AES-256 для данных в покое (at rest), TLS 1.3 для данных в транзите (in transit)
- **Многофакторная аутентификация (MFA)**: биометрия (Face ID, Touch ID), OTP-коды, push-подтверждения. MFA-системы достигают менее 0.01% компрометации защищенных аккаунтов
- **Токенизация**: замена чувствительных данных (номера карт, банковские реквизиты) на токены, что драматически уменьшает поверхность атаки
- **SOC 2 Type II**: аудит по пяти критериям доверия — безопасность, доступность, целостность обработки, конфиденциальность, приватность
- **PCI DSS**: обязательный стандарт для любых операций с платежными данными
- **Security by Design**: встраивание защиты в архитектуру с самого начала, что снижает уязвимости на 70% по сравнению с ретроактивным подходом
- **Аудит-логи**: детальное логирование всех операций с чувствительными данными

> По данным SecurityScorecard (2025), 18.4% финтех-компаний сталкивались с публично раскрытыми утечками, а 28.2% из них — с множественными инцидентами.

### Что уже есть в AnticiFi

- Офлайн-first подход: данные хранятся локально на устройстве (Isar DB), что минимизирует поверхность сетевой атаки
- JWT-аутентификация на бэкенде (NestJS + Passport)
- Bcrypt-хеширование паролей
- HTTPS-транспорт (TLS)
- Базовая структура Guards для авторизации

### Что нужно добавить (приоритизировано)

**Критический приоритет (до MVP):**

1. **Шифрование данных at rest** — зашифровать локальную базу Isar DB и серверную PostgreSQL с помощью AES-256. Финансовые данные — это наиболее чувствительная категория.
2. **Биометрическая аутентификация** — добавить Face ID / Touch ID через `local_auth` пакет Flutter для быстрого входа в приложение.
3. **Rate limiting и защита от брутфорса** — ограничить количество попыток входа, внедрить account lockout после N неудачных попыток.
4. **Валидация и санитизация входных данных** — защита от SQL injection, XSS, и других OWASP Top 10 уязвимостей на уровне бэкенда.

**Высокий приоритет (до V1.0):**

5. **Двухфакторная аутентификация (2FA)** — TOTP (Google Authenticator / Authy) как опциональный второй фактор для Pro-аккаунтов.
6. **Детальные аудит-логи** — логирование всех операций с финансовыми данными (кто, когда, что изменил).
7. **Токенизация чувствительных данных** — при хранении банковских реквизитов для Open Banking интеграций.
8. **Автоматическое завершение сессии** — logout после неактивности (настраиваемый таймер).

**Средний приоритет (до V1.5):**

9. **SOC 2 Type II подготовка** — внедрение политик и процессов для прохождения аудита (обязательно для B2B-контрактов).
10. **Penetration testing** — регулярное тестирование на проникновение сторонними экспертами.
11. **Bug bounty программа** — после достижения 100K+ пользователей.

### Конкретные рекомендации

- Использовать [flutter_secure_storage](https://pub.dev/packages/flutter_secure_storage) для хранения токенов и чувствительных данных на устройстве (Keychain на iOS, Keystore на Android)
- Внедрить certificate pinning для защиты от MITM-атак
- Для серверной стороны: использовать `helmet` middleware в NestJS для HTTP security headers
- Рассмотреть [OWASP Mobile Security Testing Guide](https://mas.owasp.org/MASTG/) как чеклист для мобильной безопасности

### Что делают лидеры рынка

**YNAB ($109/год):**
- Строгая методология zero-based бюджетирования ("4 Rules")
- Бесплатный 34-дневный триал
- Бесплатный год для студентов — стратегия привлечения будущих лояльных пользователей
- Образовательный контент и community как ключевой retention-драйвер

**Monarch Money ($99.99/год):**
- Позиционируется как главная замена Mint (закрылся в 2024)
- Bill Sync: автоматическое обнаружение рекуррентных подписок
- Инвестиционный мониторинг: аллокация, рост портфеля, доходность, диверсификация
- Goals: визуальный трекинг финансовых целей
- AI-ассистент (запущен в 2026)
- Receipt scanning (сканирование чеков)
- Рейтинг 4.8 на App Store

**Copilot Money ($95/год):**
- Только Apple-экосистема (iOS + macOS)
- AI-категоризация: персональная ML-модель для каждого пользователя, ~90% точность "из коробки"
- Адаптивное бюджетирование: AI создает и корректирует бюджеты на основе привычек
- Подключение к 10,000+ финансовым институтам
- Прогнозы перерасхода в категориях
- Чистая подписочная модель: без рекламы, без продажи данных
- Рейтинг 4.8 на App Store

### Что уже есть в AnticiFi

- Предиктивный подход — уникальное УТП, которого нет ни у одного конкурента как основной функции
- Oracle AI чат — конкуренты только начинают добавлять AI-ассистентов (Monarch в 2026)
- Управление рекуррентными платежами
- CSV импорт
- Прогноз баланса на 14/90 дней
- OCR-сканирование чеков (план V1.0)

### Что нужно добавить (приоритизировано)

**Критический приоритет:**

1. **Автоматическое обнаружение рекуррентных платежей** — как у Monarch и Copilot. Анализировать историю транзакций и автоматически предлагать добавить повторяющиеся траты.
2. **Smart категоризация транзакций** — ML-модель для автоматической классификации (как у Copilot). Начать с rule-based + merchant mapping, затем перейти к ML.

**Высокий приоритет:**

3. **Финансовые цели (Goals)** — визуальный трекинг целей (накопить на отпуск, погасить кредит) с интеграцией в прогнозы Oracle.
4. **Net worth трекинг** — агрегированный показатель благосостояния (все счета минус долги). Есть у всех конкурентов.
5. **Web dashboard** — запланировано на V1.5, но Monarch и YNAB имеют web-версии с самого начала.

**Средний приоритет:**

6. **Инвестиционный трекинг** — отслеживание портфеля (запланировано на V2.0).
7. **Семейный/парный бюджет** — Monarch активно позиционирует "couples budgeting".
8. **Образовательный контент** — финансовая грамотность как retention-инструмент (сила YNAB).

### Конкретные рекомендации

- **Ключевой дифференциатор AnticiFi — предиктивность** — нужно максимально усиливать это преимущество в маркетинге и UX. Ни один конкурент не делает прогноз будущего баланса основным экраном.
- **Ценовая стратегия ($79.99/год) дает конкурентное преимущество** — AnticiFi дешевле всех основных конкурентов при уникальном функционале.
- **Внедрить бесплатный триал 14-30 дней для Pro** — это стандарт рынка (YNAB: 34 дня, Monarch: 7 дней).
- **Приоритизировать кросс-платформенность** — Copilot теряет рынок из-за ограничения Apple-only. AnticiFi на Flutter покрывает iOS + Android сразу.

## 3. UX паттерны

### Что делают лидеры рынка

**Онбординг:**
- Поэтапный процесс с прогресс-баром (пользователи завершают онбординг значительно чаще при видимом прогрессе)
- Минимум обязательных полей при регистрации
- Social login (Apple ID, Google) для снижения барьера
- "Progressive disclosure" — запрашивать данные по мере необходимости, а не все сразу
- Демо-данные для первого запуска (чтобы пользователь сразу увидел ценность)

**Визуализация данных:**
- Подписи прямо на графиках (без необходимости угадывать)
- Подсветка аномалий и трендов: "Ваши расходы выросли на 20% по сравнению с прошлым месяцем"
- Net worth визуализация, savings goals, ежемесячные сводки
- Micro-learning: тултипы, мини-туториалы, контекстные финансовые советы в реальном времени

**Mobile-first дизайн:**
- Дашборды загружаются за <2 секунды даже на бюджетных устройствах
- Минималистичный дизайн — фокус на главном без информационного шума
- Голосовые и разговорные интерфейсы — тренд 2025-2026
- AI-driven персонализация — разный контент для разных пользовательских сегментов

**Геймификация (тренд 2025):**
- Streak-механики: "28 дней подряд в рамках бюджета!"
- Бейджи за достижения: "Savings Master", "Budget Pro"
- Прогресс-бары для финансовых целей
- Результат: **+47% retention за 90 дней** и **+48% вовлеченности** по данным Deloitte (2024)

### Что уже есть в AnticiFi

- Mobile-first архитектура (Flutter)
- Dashboard с аналитикой
- Временная шкала будущего баланса — уникальный UX-элемент
- Oracle чат — разговорный интерфейс к финансовым данным
- Push-уведомления (базовые)

### Что нужно добавить (приоритизировано)

**Критический приоритет (до MVP):**

1. **Пошаговый онбординг с прогресс-баром** — максимум 3-4 экрана:
   - Экран 1: Регистрация (email/Apple ID/Google)
   - Экран 2: Добавьте свой первый счет (ручной ввод баланса)
   - Экран 3: Укажите ближайший регулярный платеж
   - Экран 4: Ваш первый прогноз готов!
2. **Демо-режим** — возможность посмотреть приложение с тестовыми данными до регистрации. Это критично для конверсии.
3. **Скелетон-экраны и быстрая загрузка** — максимум 2 секунды для главного дашборда.

**Высокий приоритет (до V1.0):**

4. **Цветовая система рисков** — красный/желтый/зеленый для прогнозного баланса. Пользователь должен за 1 секунду понять свой финансовый статус.
5. **Контекстные инсайты на графике** — "Через 5 дней списание Netflix — баланс снизится до X". Подписи прямо на временной шкале.
6. **Тултипы и мини-туториалы** — объяснение каждого элемента интерфейса при первом использовании.
7. **Social login** — вход через Apple ID, Google.

**Средний приоритет (до V1.5):**

8. **Геймификация** — бейджи ("Первый прогноз!", "30 дней на плаву!"), streak-механика за ежедневное использование.
9. **Виджеты** — iOS Widget с текущим балансом и ближайшим прогнозом (критично для daily engagement).
10. **Dark mode** — стандартное ожидание пользователей в 2025+.
11. **Accessibility** — поддержка VoiceOver/TalkBack, достаточный контраст, масштабируемые шрифты.

### Конкретные рекомендации

- **"Time to First Value" < 60 секунд** — от скачивания до первого прогноза баланса. Это ключевая метрика для retention Day 1.
- **Использовать [fl_chart](https://pub.dev/packages/fl_chart)** для интерактивных графиков прогноза — одна из лучших библиотек для Flutter.
- **Анимации**: использовать Rive или Lottie для плавных переходов и micro-interactions (доказано увеличивают perceived quality).
- **A/B тестирование онбординга** — каждая итерация онбординга должна тестироваться через Firebase A/B Testing.

## 4. Монетизация

### Что делают лидеры рынка

**Подписочная модель (доминирует):**

| Приложение | Месяц | Год | Триал | Бесплатный tier |
|---|---|---|---|---|
| YNAB | $14.99 | $109 | 34 дня | Нет (только триал) |
| Monarch | $14.99 | $99.99 | 7 дней | Нет (только триал) |
| Copilot | $13.00 | $95 | 14 дней | Нет (только триал) |
| **AnticiFi (план)** | **$9.99** | **$79.99** | **?** | **Да (Free tier)** |

**Ключевые паттерны:**
- 97% приложений в Google Play и 95% в App Store — бесплатные. Freemium + подписка — стандартная модель
- Конверсия Free → Paid: benchmark 2-5%, топ-перформеры 6-8%
- Годовая подписка со скидкой 30-40% относительно месячной — стандарт для увеличения LTV
- Ни один из лидеров не использует рекламу — это важный сигнал доверия для финансовых приложений
- Тренд 2025-2026: гибридная монетизация (подписка + партнерские предложения)

**Тактики увеличения конверсии:**
- "Premium moment": показ premium-функции в момент, когда пользователь максимально мотивирован (например, после первого прогноза)
- Tiered pricing: 2-3 уровня для разных сегментов аудитории
- Anchoring: средний план позиционируется как "лучший выбор"
- Annual upsell: предложение перейти на годовую подписку через 1-2 месяца использования

### Что уже есть в AnticiFi

- Четкое разделение на Free и Pro тиры с логичными ограничениями
- Конкурентная цена: $79.99/год — дешевле всех основных конкурентов
- Free tier дает реальную ценность (базовый прогноз на 14 дней)
- B2B лицензирование как второй канал дохода

### Что нужно добавить (приоритизировано)

**Критический приоритет (до MVP):**

1. **Бесплатный триал Pro на 14 дней** — без привязки карты. Дать пользователю почувствовать полную ценность. Конверсия trial → paid обычно 15-25% (значительно выше, чем free → paid).
2. **Paywall UX** — продуманный экран премиум-функций с конкретными примерами ценности. Не просто "купите Pro", а "Узнайте свой баланс через 90 дней" с визуализацией.

**Высокий приоритет (до V1.0):**

3. **Годовая скидка messaging** — показывать экономию: "$9.99/мес или $79.99/год (экономия $40!)".
4. **Premium moments** — стратегическое размещение апсейл-триггеров:
   - При попытке импорта CSV → "Доступно в Pro"
   - При вопросе Oracle → "Oracle доступен в Pro. Попробуйте бесплатно 14 дней"
   - При прогнозе > 14 дней → "Хотите прогноз на 90 дней?"
5. **Интеграция платежей** — RevenueCat для управления подписками (поддерживает App Store + Google Play, стандарт индустрии).

**Средний приоритет (до V1.5):**

6. **Семейный план** — $12.99/мес за 2 пользователей. Monarch активно продвигает "couples" подход.
7. **Реферальная программа** — "Пригласите друга — получите 1 месяц Pro бесплатно". Низкий CAC при высокой конверсии.
8. **Партнерские предложения** — контекстные рекомендации финансовых продуктов (уже в плане). Начинать только после 50K+ пользователей.

### Конкретные рекомендации

- **Не запускать рекламу в приложении** — все топ-конкуренты отказались от рекламы. Для финансового приложения доверие = retention.
- **Free tier должен быть достаточно полезным**, чтобы пользователь привык и стал зависим от прогнозов. 14 дней прогноза — хороший баланс.
- **Использовать RevenueCat** — стандартная инфраструктура для подписок, A/B тестирование цен, аналитика чарна.
- **Метрики для отслеживания**: Trial-to-Paid Conversion, Monthly Churn Rate, LTV/CAC ratio (цель > 3:1), ARPU.

## 5. Интеграции (Open Banking)

### Что делают лидеры рынка

**Основные провайдеры Open Banking API:**

| Провайдер | Регион | Банков | Ключевые возможности |
|---|---|---|---|
| **Plaid** | US, Canada, UK, EU | 12,000+ | Агрегация данных, верификация, транзакции |
| **Tink** (Visa) | EU (34 страны) | 6,000+ | PSD2-compliant, Pay-by-bank, обогащение данных |
| **Salt Edge** | Global (50+ стран) | 5,000+ | Широкое покрытие, Open Banking Gateway |
| **TrueLayer** | UK, EU | 2,000+ | Открытые платежи, верификация аккаунтов |
| **MX Technologies** | US | 16,000+ | Data enrichment, analytics, verification |

**Функции, которые предоставляют провайдеры:**
- **Account aggregation**: подтягивание балансов и транзакций со всех банковских счетов
- **Transaction categorization**: автоматическая классификация транзакций
- **Data enrichment**: обогащение данных (логотипы мерчантов, категории, геолокация)
- **Payment initiation**: инициация платежей напрямую из банковского счета пользователя
- **Identity verification**: KYC/KYB через банковские данные

**Как используют конкуренты:**
- Monarch, Copilot, YNAB — все используют Plaid для подключения к банкам в US
- Подключение занимает <30 секунд
- Автоматическое обновление транзакций каждые несколько часов
- Возможность подключить банки, кредитные карты, инвестиционные счета, криптовалютные биржи

### Что уже есть в AnticiFi

- CSV импорт банковских выписок (план V1.0) — базовый способ загрузки данных
- OCR сканирование чеков (план V1.0)
- Ручной ввод транзакций
- Open Banking интеграция запланирована на V2.0 (2027)

### Что нужно добавить (приоритизировано)

**Высокий приоритет (V1.5 вместо V2.0 — ускорить!):**

1. **Plaid интеграция (US/Canada)** — это #1 запрос пользователей у всех конкурентов. Без автоматического подключения к банкам retention значительно ниже. Рассмотреть перенос с V2.0 на V1.5.
   - Стоимость: Plaid Exchange (бесплатный tier до определенного объема), далее pay-per-connection
   - Начать с бесплатного tier для валидации спроса
2. **Salt Edge или Tink (EU)** — для европейского рынка. Salt Edge покрывает 50+ стран.

**Средний приоритет (V2.0):**

4. **Transaction enrichment** — логотипы мерчантов, очистка описаний транзакций, автоматическая категоризация через API провайдера.
5. **Инвестиционные счета** — подключение к Robinhood, Vanguard, Fidelity через Plaid Investments.

**Низкий приоритет (V2.0+):**

## 6. AI в финтехе

### Что делают лидеры рынка

**Рынок AI в финтехе**: $49.43 млрд к концу 2025, CAGR 23.37%. К 2025 году 87% финансовых институтов внедрили AI-powered fraud detection.

**74% пользователей personal finance приложений предпочитают приложения с AI-инсайтами** (данные 2025).

**Ключевые AI-применения в personal finance:**

1. **Smart Transaction Categorization**
   - Copilot: персональная ML-модель для каждого пользователя, ~90% точность
   - Обучение на поведении пользователя: чем больше данных, тем точнее
   - Rule-based baseline + ML refinement — стандартный подход

2. **Anomaly Detection (Обнаружение аномалий)**
   - Выявление нетипичных трат: "Вы потратили $500 в категории 'Рестораны' — это в 3 раза больше обычного"
   - Обнаружение мошеннических операций
   - Алгоритмы: Isolation Forest, Autoencoder, LSTM-based

3. **Predictive Analytics (Предиктивная аналитика)**
   - Прогноз будущих расходов на основе исторических паттернов
   - Cashflow forecasting — ключевая функция для AnticiFi
   - Алгоритмы: Time series forecasting (Prophet, ARIMA), gradient boosting

4. **AI Chat Assistants**
   - Monarch Money: AI-ассистент (запущен 2026) — ответы на вопросы о финансах
   - Тренд: LLM-powered финансовые советники (RAG на данных пользователя)
   - Осторожность: нельзя давать финансовые рекомендации без лицензии

5. **Personalized Insights & Recommendations**
   - "Если вы сократите расходы на кофе на 30%, сэкономите $45/мес"
   - Персональные tips на основе паттернов трат
   - Оптимальное время для крупных покупок

### Что уже есть в AnticiFi

- **Oracle — предиктивный движок** — ядро продукта, анализирует историю транзакций, паттерны, регулярные платежи
- **AI чат** — диалоговый интерфейс к финансовым данным (уникальная feature)
- **Прогноз баланса** — 14/90 дней
- **Уведомления о рисках** — предупреждения до наступления проблемы

### Что нужно добавить (приоритизировано)

**Критический приоритет (до MVP):**

1. **Базовая категоризация транзакций** — начать с rule-based системы (merchant name → category mapping). База из 500-1000 правил покроет 70-80% транзакций.
2. **Simple time-series forecasting** — для MVP достаточно weighted moving average + known recurring payments. Не нужен сложный ML для старта.

**Высокий приоритет (V1.0):**

3. **ML-based категоризация** — обучить модель на данных пользователя. Подход: pre-trained model + fine-tuning на user data. Использовать scikit-learn или TensorFlow Lite.
4. **Anomaly detection** — алерты при нетипичных тратах. Начать с statistical Z-score, затем перейти к Isolation Forest.
5. **Улучшенный Oracle** — интеграция LLM (GPT-4 / Claude) для natural language understanding. RAG-архитектура на финансовых данных пользователя.

**Средний приоритет (V1.5):**

6. **Персональные инсайты** — "В этом месяце вы потратили на 25% больше на транспорт. Вот почему: [конкретные транзакции]".
7. **Savings suggestions** — "Если бы вы отменили подписку X ($14.99/мес), за год сэкономили бы $180".
8. **Spending predictions per category** — прогноз расходов по категориям, а не только общего баланса.

**Низкий приоритет (V2.0):**

9. **Investment recommendations** — ML-модели для оптимизации портфеля (требует регуляторного compliance).
10. **Advanced NLP** — голосовые запросы к Oracle ("Сколько я потратил на еду на прошлой неделе?").

### Конкретные рекомендации

- **Для MVP Oracle**: использовать LLM API (Claude/GPT) + structured prompts с контекстом финансовых данных. Не нужен custom ML — LLM справится с прогнозами и Q&A.
- **Для категоризации**: начать с [merchant category codes (MCC)](https://www.visa.com/supplierlocator/merchant/mccs.html) + keyword matching. ML добавить во V1.0.
- **Для time-series forecasting**: Facebook Prophet — простой в использовании, хорошо работает с финансовыми данными, учитывает сезонность.
- **Важно**: добавить дисклеймер "AnticiFi предоставляет информационные прогнозы, не является финансовым советником" — регуляторное требование.
- **Privacy-first AI**: по возможности запускать inference на устройстве (TensorFlow Lite) для категоризации. Серверный AI — только для Oracle чата.

## 7. Регуляторные требования

### US (Соединенные Штаты)

**Основные законы и регуляции:**

| Регуляция | Область | Применимость к AnticiFi |
|---|---|---|
| **Gramm-Leach-Bliley Act (GLBA)** | Защита финансовых данных потребителей | Высокая — обработка финансовых данных |
| **Bank Secrecy Act (BSA) / AML** | Противодействие отмыванию денег | Средняя — если будут платежные функции |
| **CCPA/CPRA** | Приватность данных (Калифорния) | Высокая — пользователи из Калифорнии |
| **SEC regulations** | Инвестиционные рекомендации | Низкая (пока нет investment advice) |
| **State lending licenses** | Кредитование | Не применимо (пока) |

**Ключевые требования:**
- Прозрачная политика обработки персональных данных
- Право пользователя на удаление данных
- Opt-in для сбора и обработки данных
- Уведомление о нарушениях безопасности (breach notification)

### EU (Европейский Союз)

**Основные законы и регуляции:**

| Регуляция | Область | Применимость к AnticiFi |
|---|---|---|
| **GDPR** | Защита персональных данных | Критическая — штрафы до €20M / 4% оборота |
| **PSD2 → PSD3 (2026)** | Платежные сервисы, Open Banking | Высокая — для bank connections |
| **DORA (2025)** | Операционная устойчивость | Средняя — IT risk management |
| **EU AI Act (2026)** | Регулирование AI-систем | Высокая — Oracle может классифицироваться как high-risk AI |
| **FiDA (новый)** | Financial Data Access | Высокая — регулирование доступа к финансовым данным |
| **AML6** | Противодействие отмыванию денег | Средняя — если будут платежные функции |

**EU AI Act — особенно важно для AnticiFi:**
- AI-системы для credit scoring, loan approval, fraud detection, AML risk profiling классифицируются как **high-risk**
- High-risk AI системы должны соответствовать строгим требованиям: risk management, human oversight, transparency, ongoing monitoring
- Oracle (предиктивный движок) может попасть под эту классификацию, если используется для финансовых рекомендаций
- **Полные обязательства вступают в силу в 2026 году**

**PSD3 (ожидается 2026):**
- Усиленные меры по предотвращению мошенничества
- Обязательные фреймворки для безопасного обмена данными между банками и третьими сторонами
- Гармонизация правил по всей Европе

### Что уже есть в AnticiFi

- Офлайн-first подход минимизирует передачу данных по сети
- Данные хранятся локально на устройстве пользователя
- Базовая JWT-аутентификация

### Что нужно добавить (приоритизировано)

**Критический приоритет (до MVP):**

1. **Privacy Policy и Terms of Service** — юридические документы, обязательные для публикации в App Store / Google Play. Включить:
   - Какие данные собираем и зачем
   - Как обрабатываем и храним
   - Права пользователя (доступ, удаление, портируемость)
   - Cookie policy (для web-версии)
2. **GDPR-compliant consent flow** — explicit opt-in для обработки данных. Нельзя использовать pre-checked boxes.
3. **Data deletion functionality** — возможность полного удаления аккаунта и всех данных (требование Apple App Store + GDPR).
4. **Дисклеймер об AI** — "AnticiFi не является финансовым советником. Прогнозы носят информационный характер."

**Высокий приоритет (до V1.0):**

5. **CCPA/CPRA compliance** — для пользователей из Калифорнии: право знать, удалить, отказаться от продажи данных.
6. **Data Processing Agreement (DPA)** — для всех third-party сервисов (Plaid, analytics, cloud).
7. **Data retention policy** — четкие правила хранения и удаления данных.
8. **Breach notification process** — план действий при утечке данных (72 часа по GDPR).

**Средний приоритет (до V1.5):**

9. **EU AI Act compliance** — документация AI-системы Oracle: описание модели, данные обучения, bias assessment, human oversight механизмы.
10. **PSD2/PSD3 compliance** — потребуется для Open Banking интеграций в EU.
11. **GLBA compliance** — safeguards rule для защиты финансовой информации (US).

**Низкий приоритет (V2.0+):**

12. **SOC 2 Type II аудит** — обязателен для B2B контрактов с банками.
13. **ISO 27001 сертификация** — международный стандарт информационной безопасности.

### Конкретные рекомендации

- **Нанять юриста с опытом в fintech** — для разработки Privacy Policy, ToS, и оценки регуляторных рисков. Это критично и не должно делаться самостоятельно.
- **Privacy by Design** — встроить приватность в архитектуру: минимизация данных, шифрование, псевдонимизация.
- **Data mapping** — создать карту всех данных: что собираем, где храним, кому передаем, как долго храним.
- **Для EU AI Act**: предусмотреть "explainability" для прогнозов Oracle — пользователь должен понимать, почему AI предсказал определенный баланс.
- **Регулярно проводить DPIA (Data Protection Impact Assessment)** — обязательно по GDPR для обработки финансовых данных.

## Ключевые выводы

1. **Уникальное преимущество AnticiFi — предиктивный подход — подтверждается исследованием**. Ни один конкурент не делает прогноз будущего баланса основной функцией. Это реальный blue ocean.

2. **Безопасность — обязательный минимум**. Шифрование данных и биометрическая аутентификация должны быть в MVP. 18% финтех-компаний сталкивались с утечками — это неприемлемый риск для финансового приложения.

3. **Bank connections (Plaid) — ключевой фактор retention**. Рекомендуется перенести интеграцию с V2.0 на V1.5. Ручной ввод создает слишком высокий барьер для постоянного использования.

4. **Ценовая стратегия конкурентоспособна**. $79.99/год — дешевле YNAB ($109), Monarch ($99.99) и Copilot ($95). Наличие Free tier — дополнительное преимущество.

5. **AI Oracle — второй ключевой дифференциатор**. Конкуренты только начинают внедрять AI-ассистентов (Monarch в 2026). AnticiFi имеет шанс быть первым с полноценным AI-chatbot для personal finance.

6. **Регуляторика требует внимания с первого дня**. Privacy Policy, GDPR consent, и data deletion — обязательны для публикации в App Store. EU AI Act (2026) может затронуть Oracle.

7. **Геймификация = retention**. Приложения с элементами геймификации показывают +47% retention за 90 дней. Внедрение базовых элементов (streaks, badges) — quick win для V1.5.

## Проблема

Большинство людей не знают своего реального финансового будущего.

Существующие финансовые приложения — Mint, YNAB, CoinKeeper и десятки их аналогов — решают одну задачу: они помогают понять, куда уже ушли деньги. Это ретроспективный анализ. Пользователь вводит расходы, видит категории трат за прошлый месяц и получает ответ на вопрос "где я был?". Но не на вопрос "где я окажусь?".

### Конкретные боли пользователей

**1. Неожиданный минус на счёте.**
Человек тратит деньги в течение месяца, не зная, что в конце его ждёт автоплатёж за аренду, страховку и три подписки одновременно. В результате — овердрафт или вынужденный долг.

**2. Невозможность планировать покупки.**
"Могу ли я купить ноутбук на следующей неделе, не выйдя в минус до зарплаты?" — ни одно существующее приложение не даёт ответа на этот вопрос автоматически.

**3. Реактивное, а не проактивное управление.**
Пользователи узнают о проблеме после того, как она произошла. Приложение показывает: "В этом месяце ты потратил на 30% больше, чем планировал." Но уже поздно.

**4. Фрилансеры и нестабильный доход.**
Для людей с нерегулярными доходами управление денежным потоком критически сложно. Традиционные инструменты бюджетирования построены на допущении фиксированного ежемесячного дохода и практически бесполезны для фрилансеров.

**5. Психологическая нагрузка.**
Постоянная неопределённость относительно финансового состояния создаёт хроническое беспокойство. Исследования показывают, что финансовый стресс — один из главных источников тревожности у людей 25–40 лет.

> **Ключевой инсайт:** Люди не нуждаются в ещё одном способе смотреть назад. Им нужен инструмент, который смотрит вперёд.

## Решение

AnticiFi — первое **предиктивное** приложение для личных финансов. Его ядро — AI-движок "Oracle", который отвечает на один вопрос:

> **"Каким будет мой баланс на дату X?"**

### Как это работает

```
Пользователь             AnticiFi                      Oracle (AI)
     |                       |                               |
     |-- "Что будет          |                               |
     |    25 апреля?" ------>|                               |
     |                       |-- история транзакций -------->|
     |                       |-- регулярные платежи -------->|
     |                       |-- паттерны трат ------------->|
     |                       |-- ожидаемые доходы ---------->|
     |                       |                               |
     |                       |<-- прогноз баланса: 12,400р --|
     |<-- "12,400 рублей" ---|                               |
     |    "Риск: подписки    |                               |
     |     -3,200 р." -------|                               |
```

### Ключевые компоненты решения

**1. Oracle — предиктивный движок**

Движок анализирует:
- Историю транзакций (паттерны трат по дням недели, числам месяца, сезонность)
- Запланированные регулярные платежи (аренда, кредиты, подписки)
- Ожидаемые поступления (зарплата, фриланс-платежи, переводы)
- Аномалии и исключения (нетипичные траты, отмены подписок)

Результат — прогноз баланса с указанием уровня уверенности и перечнем рисков.

**2. Умный импорт данных**

Пользователь может добавить данные тремя способами:
- **CSV-импорт** банковских выписок (поддержка форматов крупных банков)
- **OCR-сканирование чеков** через камеру телефона (Google ML Kit)
- **Ручной ввод** транзакций и регулярных платежей

**3. Управление регулярными платежами**

Пользователь один раз указывает:
- Аренду: 15 000 р., каждое 1-е число
- Netflix: 999 р., каждое 17-е число
- Интернет: 600 р., каждые 30 дней

Oracle автоматически учитывает их во всех прогнозах. При приближении платежа — уведомление заранее.

**4. Временная шкала будущего**

Вместо графика прошлых расходов — интерактивный график будущего баланса. Пользователь видит:
- Прогнозируемую кривую баланса на 30/60/90 дней вперёд
- Маркеры запланированных списаний
- Зоны риска (периоды, когда баланс может уйти в минус)
- Ожидаемые поступления

**5. Офлайн-первый подход**

Критические данные и последний актуальный прогноз хранятся в локальной базе данных (Isar DB) на устройстве пользователя. Приложение полностью функционально без подключения к интернету. Синхронизация происходит при восстановлении соединения.

Подробнее о технической реализации — в [Архитектуре системы](./architecture.md) и [Мобильном приложении](./mobile-app.md).

## Целевая аудитория

### Первичная аудитория: "Работающие и считающие"

**Молодые профессионалы, 25–40 лет**, живущие от зарплаты до зарплаты в психологическом смысле — даже при достаточном доходе они испытывают тревогу из-за непредсказуемости финансов.

Характеристики:
- Доход: средний и выше среднего (50 000 – 200 000 р./мес.)
- Регулярно пользуются мобильным банкингом
- Имеют 3–7 активных подписок (стриминг, SaaS, фитнес)
- Технически грамотны, готовы доверять AI-инструментам
- Основная боль: внезапные расходы нарушают финансовые планы

**Размер аудитории (Россия + СНГ):** ~15 миллионов человек.
**Готовность платить:** высокая — при условии очевидной ценности.

### Вторичная аудитория: Фрилансеры и самозанятые

**Фрилансеры с нерегулярным доходом**, для которых прогнозирование денежного потока — вопрос выживания бизнеса.

Характеристики:
- Доход нестабильный, приходит "волнами"
- Сложно планировать крупные расходы
- Часто работают с несколькими источниками дохода
- Нуждаются в понимании: "Когда мне платить налоги и хватит ли денег?"

**Ключевая ценность для них:** прогноз кассового разрыва и рекомендации по оптимальному времени крупных трат.

### Третичная аудитория: Малый бизнес (личные финансы владельца)

**Владельцы малого бизнеса**, которым нужен контроль над личными финансами отдельно от бизнес-счетов.

Эта аудитория — потенциальный мост к B2B-продукту: если AnticiFi докажет ценность для личных финансов, следующий шаг — корпоративная версия для cash flow forecasting малого бизнеса.

### B2B-аудитория: Банки и финтехи

**Финансовые учреждения**, которым нужен предиктивный AI-модуль для встраивания в собственные продукты.

Характеристики:
- Региональные банки, необанки, финтех-стартапы
- Ищут готовое AI-решение вместо разработки с нуля
- Ценят: white-label возможности, соответствие регуляторным требованиям, быстрое внедрение

Подробнее о пользовательских сценариях для каждого сегмента — в [Пользовательских историях](./user-stories.md).

## Конкурентные преимущества

### Анализ конкурентов

| Продукт | Подход | Слабость |
|---|---|---|
| Mint (Intuit) | Ретроспективный учёт расходов | Только прошлое, нет прогнозов |
| YNAB | Бюджетирование на основе правил | Требует ручного планирования каждой категории |
| CoinKeeper | Ручной учёт расходов | Ретроспективный, нет AI |
| Personal Capital | Инвестиции + нетворт | Не фокусируется на ежедневном балансе |
| Copilot | Умная категоризация | Только США, нет предиктивности |

**Вывод:** Ни один из существующих продуктов не строит предиктивный прогноз баланса как основную функцию.

### Наши преимущества

**1. Предиктивный подход как основа, а не дополнение**

AnticiFi проектировался с нуля как predictive-first продукт. Прогноз баланса — это не фича в меню, это главный экран приложения. Конкурентам трудно скопировать это, потому что для них это означало бы переосмысление всей продуктовой философии.

**2. OCR-сканирование чеков**

Встроенное сканирование чеков через камеру телефона снижает барьер входа: пользователю не нужно ждать банковской выписки, чтобы добавить расход. Это критично для тех, кто платит наличными.

**3. Oracle — диалоговый интерфейс к финансовым данным**

Пользователь может задать вопрос в свободной форме: "Хватит ли мне денег до зарплаты, если я куплю телефон за 40 000?" — и получить конкретный ответ с объяснением. Это принципиально иной UX-паттерн.

**4. Офлайн-первый с защитой приватности**

В эпоху утечек данных возможность хранить финансовую информацию локально — конкурентное преимущество. Мы обрабатываем данные на устройстве там, где это возможно.

**5. Уведомления о риске до наступления события**

"Через 5 дней ожидается списание 15 000 р. за аренду. По текущей динамике ваш баланс составит 9 200 р. Рекомендуем пополнить счёт." — это сообщение появляется до проблемы, а не после.

## Бизнес-модель

### Freemium для B2C

#### Free — бесплатно

Цель: снизить барьер входа до нуля, дать пользователю почувствовать ценность продукта.

Включает:
- 1 банковский счёт
- Ручной ввод транзакций
- Базовый прогноз баланса на 14 дней
- До 5 регулярных платежей
- Базовые уведомления

Ограничения:
- Нет OCR-сканирования
- Нет CSV-импорта
- Нет диалогового Oracle
- Прогноз только на 14 дней (не 90)

#### Pro — $9.99 в месяц / $79.99 в год

Цель: монетизировать пользователей, получивших реальную ценность от бесплатной версии.

Включает всё из Free, плюс:
- Неограниченное количество счетов
- OCR-сканирование чеков
- CSV-импорт банковских выписок
- Прогноз на 90 дней
- Неограниченные регулярные платежи
- Диалоговый Oracle (AI-запросы)
- Детальная аналитика паттернов трат
- Приоритетная поддержка
- Экспорт данных

**Экономика подписки:**
- Целевая конверсия Free → Pro: 5–8%
- При 100 000 пользователях и 6% конверсии: 6 000 Pro-пользователей
- MRR: $59 940 (~$720K ARR)

### B2B лицензирование

White-label версия AnticiFi для банков и финтех-компаний.

**Модель ценообразования:**
- Фиксированная лицензия + revenue share
- Ориентировочно: $50 000–$500 000/год в зависимости от размера организации
- Или: $0.50–$2.00 за активного пользователя/месяц

**Ценность для партнёра:**
- Готовый AI-модуль без затрат на R&D
- Повышение retention пользователей банковского приложения
- Новый дифференциатор на рынке

### Партнёрские предложения

Монетизация через рекомендацию финансовых продуктов партнёров.

Принцип: AnticiFi видит финансовую ситуацию пользователя. Если пользователь регулярно держит остаток выше 50 000 р. — предложить накопительный счёт с выгодной ставкой. Если часто уходит в минус — предложить кредитную линию с хорошими условиями.

**Важно:** Рекомендации только тогда, когда они действительно релевантны. Агрессивная реклама разрушает доверие — основной актив финансового приложения.

**Модели монетизации партнёрств:**
- CPA (cost per acquisition): фиксированная выплата за каждого привлечённого клиента
- Revenue share от комиссий продуктов партнёра

Подробнее о технических требованиях к каждой из моделей — в [Техническом задании](./technical-spec.md).

## Стратегия выхода

### Фаза 1: Product-Market Fit (0–18 месяцев)

**Цель:** Найти сегмент пользователей, для которых AnticiFi — незаменимый инструмент.

Метрики успеха:
- 10 000+ активных пользователей (DAU > 30%)
- NPS > 50
- Retention 30 дней > 40%
- Первые Pro-пользователи с органической конверсией

**Стратегия роста:**
- Органический рост через word-of-mouth (финансовые community, Reddit, Telegram)
- Content marketing: статьи о предиктивном управлении финансами
- ASO (App Store Optimization)
- Партнёрства с финансовыми блогерами

### Фаза 2: Масштабирование (18–36 месяцев)

**Цель:** Доказанная бизнес-модель, рост к $1M ARR.

Требования для привлечения Series A:
- 100 000+ пользователей
- $500K+ ARR
- Чёткая unit economics (LTV > 3x CAC)
- Первые B2B-контракты

**Использование инвестиций:**
- Расширение команды (ML-инженеры, продакты, маркетинг)
- Географическая экспансия (Европа, Латинская Америка)
- Развитие B2B-направления

### Фаза 3: Выход (36+ месяцев)

**Вариант A: Стратегическое приобретение**

Наиболее вероятные покупатели:
- **Финансовые агрегаторы:** Plaid, Yodlee, MX Technologies — им нужен предиктивный AI-слой поверх агрегированных данных
- **Крупные банки:** ВТБ, Сбер, Тинькофф (в России); Chase, Wells Fargo (глобально) — для встраивания в собственные приложения
- **Финтех-платформы:** Revolut, N26, Wise — для усиления функциональности personal finance
- **Крупные tech-компании:** Apple (интеграция с Apple Pay / Wallet), Google (Google Pay)

**Оценка при приобретении:**
- 10x ARR — стандартный мультипликатор для SaaS финтех
- При $10M ARR → оценка $100M+
- При уникальном AI IP → премия 30–50%

**Вариант B: IPO**

Условия для рассмотрения:
- 10M+ активных пользователей
- $50M+ ARR
- Устойчивый рост 50%+ year-over-year
- Позиция лидера рынка predictive personal finance

IPO позволит сохранить независимость и монетизировать рост через публичные рынки.

**Вариант C: Продолжение роста**

Если ни приобретение, ни IPO не оптимальны на данном этапе — фокус на достижении положительного cash flow и построении устойчивого прибыльного бизнеса.
